# Serverless AI Gateway - 前端开发 Roadmap

## 概述

本文档描述 Serverless AI Gateway 前端管理界面的开发路线图，分为 3 个主要版本逐步实现。

---

## 版本规划总览

| 版本 | 阶段 | 目标 | 预计周期 |
|------|------|------|----------|
| v1.0 | MVP 第一部分 | 核心框架 + 用户管理 | 1 周 |
| v1.1 | MVP 第二部分 | 供应商 + 模型管理 | 1-2 周 |
| v2.0 | 监控分析第一部分 | 请求记录 | 1 周 |
| v2.1 | 监控分析第二部分 | 仪表盘 + API 测试 | 1 周 |
| v3.0 | 增强体验第一部分 | 删除 + 导出 | 1 周 |
| v3.1 | 增强体验第二部分 | 响应式 + 主题 + 性能 | 1 周 |

---

## v1.0 - MVP 第一部分

### 版本目标

**完成基础框架搭建和用户管理功能，建立项目骨架和认证体系。**

### 功能清单

#### 1.1 登录认证（P0）
- Token 登录表单
- 登录状态验证
- Token 存储管理（localStorage）
- 登出功能

#### 1.2 基础框架（P0）
- Vue 3 + TypeScript + Vite 项目搭建
- Ant Design Vue 集成
- Vue Router 路由配置
- Pinia 状态管理配置
- Axios 封装（请求/响应拦截器）
- 全局错误处理
- 侧边栏 + 主内容区域布局
- 顶部导航栏

#### 1.3 用户管理（P0）
- 用户列表展示（表格）
- 新建用户功能
- 用户详情查看
- Token 显示/隐藏切换
- 用户类型筛选（normal/admin）
- 按用户名搜索
- 分页功能

#### 1.4 系统状态（P1）
- 系统运行状态展示
- 简单的状态卡片

### 验收标准

- 管理员可以通过 Token 成功登录
- 可以创建、查看用户
- 表单验证正常工作
- 用户列表支持搜索、筛选、分页
- 无明显 Bug，核心流程可正常使用

### 交付物

- 可运行的前端项目
- 登录认证功能
- 用户管理功能

## v2.1 - 监控分析第二部分

### 版本目标

**增加仪表盘统计和 API 测试工具，提升数据可视化和开发调试体验。**

### 功能清单

#### 2.2 仪表盘增强（P0）
- 今日请求总数统计
- 今日请求成功率统计
- 活跃用户数统计
- 活跃模型数统计
- 最近请求快捷入口
- 数据自动刷新（30秒）
- 刷新控制按钮

#### 2.3 API 测试工具（P0）
- API 格式选择器（OpenAI/Anthropic）
- 模型选择器
- 消息输入区域（支持多轮对话）
- 参数配置面板（temperature、max_tokens 等）
- 发送按钮
- 响应展示区域
- 流式响应支持（SSE）
- 清除/重新发送功能
- 测试历史记录

#### 2.4 用户体验优化（P1）
- 错误提示优化
- 加载动画优化

### 验收标准

- 仪表盘统计数据准确
- 数据自动刷新正常
- API 测试工具可用
- 流式响应展示正常

### 交付物

- 增强版仪表盘
- API 测试工具

## 优先级说明

| 优先级 | 说明 | 处理原则 |
|--------|------|----------|
| P0 | 必须实现 | 当前版本必须完成，阻塞发布 |
| P1 | 重要功能 | 当前版本尽量完成，可推迟 |
| P2 | 增强功能 | 可推迟到后续版本 |

## 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 后端 API 接口变更 | 中 | 提前确认接口规范，保持沟通 |
| UI 组件库兼容性问题 | 低 | 选用稳定的组件库版本 |
| 性能问题（大量数据） | 中 | 使用虚拟滚动、分页加载 |
| 移动端适配复杂度高 | 低 | 优先保证桌面端体验 |
| 开发进度延期 | 中 | P2 功能可推迟到下个版本 |
